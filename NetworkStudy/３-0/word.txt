 

■ ネットワーク層
　　通信プロトコルの機能や役割を構成する層
　　単一の、あるいは相互接続された複合的なネットワークの上で末端から末端までデータを贈り届ける役割を担う

　IP
　 （Internet Protocol）
　　相手先のアドレス情報をもとにしてパケットを送信する機能
　　インターネット層のプロトコルとなる

　IPヘッダ
　　IPにおけるデータの送受信単位であるIPデータグラム（パケット）の先頭にある、
　　一定の形式に従って制御情報などを記載した部分のこと。

　ICMP
　 （Internet Control Message Protocol）
　　IP通信の疎通状態を調査したり、正常に通信できない場合に
　　送信元ホストへエラー情報を返したりする、IP通信を補佐するためのプロトコル

　ICMPメッセージ
　　ICMPの返答メッセージで「Queryメッセージ」と「Errorメッセージ」の2つに分けられる
　　Queryメッセージは、あるノードから特定のノードに対する通信状態を確認する、いわば「調査・問い合わせメッセージ」を指し、Errorメッセージは、IP通信において何らかの問題が発生したときに送信元ホストへ知らせる「エラー通知メッセージ」を指す

　IPアドレス
　　スマホやPCなど、ネットワーク上の機器に割り当てられるインターネット上の住所のような存在

　　オクテット
　　　8ビットの情報量を表す単位

　　ネットワーク部
　　　IPアドレスが属しているネットワークを識別するための部分

　　ホスト部
　　　IPアドレスが属しているネットワークを識別するための部分

　　クラス
　　　ネットワークの規模に応じてABCDに分けられている

　　プライベートIPアドレス
　　　会社、家庭などの組織内で割り当てられるIPアドレス

　　グローバルIPアドレス
　　　インターネットに接続された機器に割り当てられるIPアドレス

　ネットワークアドレス
　　ネットワーク自身に割り当てられたアドレス

　ブロードキャストアドレス
　　ネットワーク内のすべての機器にデータを一斉配信するために使われる特殊なアドレス

　ループバックアドレス
　　コンピュータ自身を示すアドレスのこと

　サブネット
　　機器やアドレスの数が多い大きなネットワークを管理しやすいよう小さく分割したネットワーク


　サブネットマスク
　　IPアドレスの先頭から何ビットをネットワークアドレスにするか定義する32ビットの数値

　　プレフィックス表記
　　　特定の意味を持って指定した数字


　デフォルトゲートウェイ
　　コンピュータや通信機器などのネットワーク設定。送信相手がわからない時にとりあえずデータを送信する中継機器のアドレス

　IPv6
　　０〜Fの１６進数で表示されるアドレス

　　サブネットプレフィックス
　　　IPv4におけるネットワーク部に相当する
　　　基本的に前半64ビットがサブネットプレフィックスにあたる
　　　　例）2001 : db8 : abcd : 0012 : : 0 / 64
　　　　サブネット：2001 : db8 : abcd : 0012 : 0000 : 0000 : 0000 : 0000
　　　　　　　 　~  2001 : db8 : abcd : 0012 : ffff : ffff : ffff : ffff

 

　　インターフェイスID
　　　IPv4におけるホスト部に相当します。
　　　通常のIPv6ノードは、インタフェイスIDをMACアドレスから算出します。
　　　この計算の方式には、EUI-64形式が利用されます。
　　　　ーMACアドレス6バイトの上位3バイトと下位3バイトを分け、間に"fffe"を入れる
　　　　ー上位1バイト目の下位2ビット目を"1"にする
　　　　例）MACアドレス："0080.bdcf.f100" だった場合、
　　　　　上位3バイト（"0080bd"）と下位3バイト（"cff100"）の間に"fffe"を入れた
　　　　　"0080bdfffecff100"の上位1バイト目の下位2ビット目を"1"にした、
　　　　　"0280bdfffecff100"がこのインタフェイスのインタフェイスIDになります。

・ルータ

　　コンソール接続
　　　大型機械や電子機器などの操作や監視の為の装置を一箇所に集めた装置のこと

　　VTY接続
　　　バーチャルテレタイプ。ルータなどの通信機器にネットワークを通じて別のコンピュータから接続し、操作や管理を行うための仮想的な接続口

　　AUX接続
　　　ルータなどのネットワーク機器に管理用としてあるポートの一つ。他にコンソールポートなどがある

　　ルーティングテーブル
　　　ルーターにある経路表

　　　ネクストホップ
　　　　ルーティングテーブルにある、次に転送する隣接ルーターのこと

　　ARPテーブル
　　　IPアドレスとMACアドレス（EthernetやWi-Fiなどの物理アドレス）の対応表

　　　ARP
　　　　IPアドレス→MACアドレスを解決します

　　コアルータ
　　　ルータの役割で基幹ネットワークの中心部で用いられるものをコアルータという

　　センタールータ
　　　遠距離通信を挟んで中心部と末端側の両方の装置を示す

　　エッジルータ
　　　遠距離通信の末端側を示す

　　ブロードバンドルータ
　　　通信事業者のブロードバンド回線を通じてインターネットなどを利用する際に用いるルータ

 

・ルーティング

　　直接接続
　　　ルータと隣接しているネットワークを自動で登録する方法

　　スタティックルーティング
　　　静的経路選択　ルータが管理者の設定した経路表に従って固定的に経路選択を行うこと

　　ダイナミックルーティング
　　　動的経路選択　経路情報をルータ同士が交換し合い自動的に経路生成、更新し続ける

　　　ディスタンスベクタ型
　　　　ルーティングの分類の一つ　隣接ルータで情報を交換し合いどの隣接ルータを経由すれば最短のホップ数で宛先に届くかを基準に経路指定する方式　RIPやIGRPが採用している

　　　リンクステート型
　　　　ルーティングプロトコルの特徴・分類の一つで、どのルータとどのルータが隣接しているかという接続情報（リンクステート）を交換し合い、この情報の集合に基づいて経路を選択する方式。

　　　ハイブリッド型
　　　　IGRPを拡張したもの。IGPの一つ。


 

■ トランスポート層

・コネクション型通信
　　コネクション型通信　データ通信において通信を開始する前に相手との間で仮想的な専用経路（コネクション）を確率し、それを通じてデータの送受信を行う通信方式

　　ACK
　　　二者の通信間で相手方に何等かの肯定的な応答を返す際に信号やデータ、パケットなどのこと。データが正しく受信できた　といった内容を伝達するために用いられる

・コネクションレス型通信
　　相手とのデータ通信前に仮想的な専用通信路（コネクション）を確率し、それに応じてデータの送受信を行う通信方式。送受信の確実性が保証される

　　TCP
　　　インターネットなどのIPネットワークでIPの上位層であるトランスポート層のプロトコル（通信規約）として標準的に用いられるものの一つ　信頼性は高いが即時性や高速性は得られにくい

　　スリーウェイハンドシェイク
　　　IPネットワーク上でTCPによるコネクションを確立する為の手順。パケットの送受信を3回行う

　　Telnet
　　　IPネットワークを通じて遠隔操作する為の通信規約


　　HTTP
　　　ウェブサーバとwebクライアントの間で送受信を行うために用いられるプロトコル。HTMLやスタイルシート、スクリプト、画像、音声、動画などのメタ情報を含めてやりとりすることができる

　　FTP
　　　ファイル転送プロトコル。TCP\IPネットワークでファイル転送を行うプロトコルの一つ

　　SMTP
　　　シンプルメールトランスファープロトコル。電子メールを送信、転送するコンピュータのこと

・TCPヘッダ
　　TCPセグメントの先頭部分にある領域。制御情報を記述してる

・UDP
　　ユーザーデータグラムプロトコル。トランスポート層で使用されるプロトコルの一つ。低遅延だが信頼性は低い

　　RTP
　　　リアルタイムトランスポートプロトコル　そのままの意味　音声や動画などのデータストリームをリアルタイムに配送するためのデータ通信プロトコルである

　　SNMP
　　　シンプルネットワークマネージメントプロトコル　IPネットワーク上の機器をネットワーク経由で遠隔制御、監視、管理する

　　DHCP
　　　ダイナミックホストコンフィギュレーションプロコトル　ネットワークに新たに接続した機器にIPアドレスなど必要な情報を自動で割り当てる

　　TFTP
　　　トライバルファイルトランスファープロトコル　ファイル転送、自動配布を行うためのプロトコル

　　RIP
　　　ラスターイメージプロセッサ。文書データや画像データを解析しプリンタで印刷可能なビットマップに変換するソフトウェアのこと

・UDPヘッダ
　　UDPデータの先頭部分にある制御情報などをまとめた部分
